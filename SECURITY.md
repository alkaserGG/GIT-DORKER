# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in GitDorker-Elite, please report it responsibly.

**Do not open a public GitHub issue.** Instead, send an email to:

📧 **abdoalkaser53@gmail.com**

Include the following details:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and provide a timeline for resolution.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Security Best Practices

- Never commit your `.env` file or hardcode tokens.
- Always use the tool only on domains you own or have explicit permission to test.
- Review the tool's output before sharing to avoid accidental exposure of sensitive data.
